Phase 1 — Fondations v0.9 beta

Infrastructure

Page d'accueil index.html

Choix prof / élève, badges souveraineté (RGPD, Nuage, Albert, Gratuit)

Déploiement GitLab Pages (Forge)

Pipeline CI/CD fonctionnel, fichiers servis en HTTPS sur la Forge des Communs Numériques Éducatifs

Workflow n8n — Experia Storage

3 nœuds (Webhook → Code → Répondre), proxy WebDAV avec gestion CORS, actions test/list/read/write/delete, auto-création dossier /experia/, manifest.json

Stockage Nuage (WebDAV Nextcloud)

Sessions et résultats stockés dans /experia/ sur le compte Nuage du professeur. Serveur nuage03.apps.education.fr (valeur par défaut corrigée)

Intégration Albert API via proxy n8n (Mentoria)

Réutilisation du proxy Albert de Mentoria, paramétrage modèle + clé API par le professeur

Interface professeur (prof.html)

Écran connexion Nuage

Saisie serveur, login, mot de passe d'application. Test de connexion avant toute opération.

Tableau de bord sessions

Liste des sessions existantes lues depuis le Nuage, accès aux résultats

Création de session

Titre, saisie corpus texte libre, upload fichier TXT/PDF, configuration du nombre de questions par activité, choix modèle Albert. Compatible avec la transcription d'une vidéo YouTube (copier-coller depuis YouTube → Afficher la transcription).

Génération IA (6 activités)

6 appels Albert successifs avec prompts spécialisés, barre de progression, sauvegarde JSON sur Nuage

Lien de session élève

Lien eleve.html#config=BASE64(JSON) avec credentials + sessionId encodés, bouton copier

Lien de connexion rapide Prof

Génération d'un lien prof.html#login=BASE64 pour une reconnexion instantanée sur n'importe quel poste

Tableau de résultats (Live)

Lecture de tous les result_*.json, auto-polling (30 s), indicateur de rendu définitif (submitted), affichage des points réels sur barème.

Sélection des activités par cartes

Interface redessinée en grille 2 colonnes — chaque carte est entièrement cliquable pour activer/désactiver l'activité. Le champ nombre de questions reste éditable sans déclencher le toggle.

Écran de validation avant publication

Après génération, le prof visualise toutes les questions, peut en supprimer individuellement, ajuster le barème par activité, puis publier le lien élève en connaissance de cause.

Export évaluation papier (Word)

Bouton "Évaluation papier" dans l'écran de session — génère un document Word (.doc) avec les questions formatées et un tableau de correction prêt à imprimer.

Interface élève (eleve.html) — 8 activités

QCM — Questions à choix multiples

4 propositions, feedback immédiat avec explication, correction automatique

Paires — Associer les termes

Cliquer pour sélectionner puis associer, pénalité de points par erreur

Mots Mêlés — Grille dynamique

Génération de grille, sélection par clic (1ère et dernière lettre)

Mots Croisés — Grille interactive

Croisement automatique des mots, définitions dynamiques, validation visuelle

Rapidité — QCM chronométré

10 secondes par question, score bonus proportionnel au temps restant

Compléter — Textes à trous

Un trou par phrase, tolérance orthographique (distance de Levenshtein, 20%)

Intrus — Trouver l'affirmation fausse

4 affirmations dont une fausse, correction automatique par index

Questions IA — Réponses libres

Réponse en texte libre, correction par Albert (score 0-10 + feedback personnalisé)

Workflow de rendu & Reprise

Boutons "Rendre le travail" (verrouillage) et "Continuer plus tard" (sauvegarde partielle), gestion automatique de la reprise au retour

Synthèse finale — score en points ET en %

Récapitulatif affiché en X / Y pts et Z %, soumission vers le Nuage

Points partiels conservés si départ en cours d'activité

savePartialScore() capte les points acquis dans une activité interrompue. rawMax corrigé pour paires, mots mêlés, mots croisés, rapidité, questions IA. Affichage orange ~ XX% sur le tableau de bord élève.

Interface professeur — nouvelles fonctionnalités (v1.3)

Persistance session / auto-login

Les 4 credentials (serveur, login, mot de passe d'application, clé Albert) sont mémorisés dans localStorage. Auto-login au rechargement si tous présents.

Bouton déconnexion

Icône en haut à droite du tableau de bord — vide le localStorage et retourne à l'écran de connexion. Essentiel pour ordinateurs partagés.

Écran de bienvenue — première connexion

Si le manifest Nuage n'existe pas, un écran de bienvenue explique le fonctionnement. Le bouton "Démarrer" crée experia_manifest.json et experia_library.json dans le Nuage puis ouvre le tableau de bord.

Toggle "Afficher la correction" dans les paramètres de session

Case à cocher dans l'onglet Paramètres de l'écran de session — modifiable après publication. Met à jour showCorrection dans le fichier session sur le Nuage.

Mes évaluations — dossiers & classement

Onglet "Mes évaluations" avec dossiers personnalisés. Classer une session dans un dossier (retire du tableau de bord, range dans experia_library.json). Créer une session de classe depuis un dossier (réutilise les questions, crée une nouvelle entrée dans le manifest).

Export CSV des résultats élèves

Bouton d'export dans l'écran de résultats — génère un fichier CSV avec identifiant élève, score par activité, total, date de rendu.

En cours — À valider en test

Test du flux complet points partiels Priorité 1

Ouvrir une activité, répondre partiellement, cliquer Retour → vérifier l'affichage orange ~ XX% sur le tableau de bord élève. Tester "Rendre le travail" avec des activités partielles. Vérifier que le total côté prof inclut ces points.

Test dossiers / Mes évaluations

Classer une session dans un dossier → vérifier qu'elle disparaît du tableau de bord. Créer une session depuis le dossier → vérifier qu'elle apparaît avec un lien élève valide. Supprimer un dossier.

Test première connexion (welcome screen)

Se connecter avec un compte Nuage sans fichier experia_manifest.json → vérifier l'affichage du welcome screen. Cliquer "Démarrer" → vérifier que les deux fichiers sont créés dans le Nuage.

Validation du jeu Paires sur mobile

L'interaction cliquer-pour-associer doit être testée sur écran tactile (touch events).

Validation de la lecture PDF

L'upload PDF est implémenté mais doit être testé avec un vrai fichier PDF en production.

À faire — Améliorations prochaines

Remplacer prompt() dans duplicateForClass() Priorité haute

La duplication de session utilise encore un prompt() natif peu élégant. À remplacer par une vraie modale avec champ de saisie avant la généralisation à d'autres profs.

QR code du lien élève

Générer un QR code à projeter au tableau ou scanner depuis un smartphone. Avertissement si la session contient des mots mêlés ou mots croisés (peu adaptés au petit écran).

Expiration de session

Champ "date de fermeture" à la création — après cette date, les élèves ne peuvent plus soumettre. Vérification côté client à la lecture de la session.

Mise à jour du guide professeur

guide_experia.html et setup_experia.html datent de l'ancienne architecture. À mettre à jour ou supprimer — setup_experia.html n'est plus pertinent (plus de compte à créer, juste s'authentifier avec les credentials Nuage).

Corrections cosmétiques texte élève

Écran fermée : "session" → "évaluation". Écran connexion : "Rejoindre une session" → "Rejoindre une évaluation".

Vision future — Post v1.0

Idées d'évolution à long terme, non planifiées.

Gestion multi-classes

Associer un groupe / une classe à une session. Résultats organisés par classe dans le tableau de bord.

Mode révision / relance

Permettre à l'élève de rejouer une session, avec présentation prioritaire des questions ratées. Score différentiel affiché.

Différenciation pédagogique

Générer plusieurs niveaux de difficulté pour un même corpus. Le lien de session intègre le niveau cible.

Analytics de classe

Visualisation graphique des résultats : distribution des scores, taux de réussite par activité, questions les plus souvent ratées.

Bibliothèque de corpus partagés

Espace de partage entre enseignants : un prof publie un corpus, d'autres peuvent l'utiliser pour générer leur propre session.

Sécurisation des credentials (clé API côté serveur)

Stocker la clé API Albert dans n8n (côté serveur) pour ne plus l'exposer dans l'URL. Nécessite une évolution du workflow n8n.

Nouveaux types d'activités

Chronologie à remettre dans l'ordre, carte mentale à compléter, vidéo + questions, dictée automatique...
